# 명령어의 구조

## 연산 코드와 오퍼랜드
* 명령어는 연산 코드와 오퍼랜드로 구성되어 있다.
명령어가 수행할 연산을 **연산 코드**라 하고, 연산에 필요한 데이터가 저장된 위치를 **오퍼랜드**라고 한다.
혹은 연산 코드는 연산자, 오퍼랜드는 피연산자라고 부르기도 한다.

## 오퍼랜드
* 오퍼랜드가 담기는 영역을 오퍼랜드 필드라고 한다.
* 오퍼랜드의 필드에는 숫자나 문자 등을 나타내는 데이터 또는 메모리나 레지스터 주소가 올 수 있다.
* 오퍼랜드 필드에는 보통 데이터를 직접 명시하기 보다는 메모리 주소나 레지스터 이름을 담기며, 그래서 오퍼랜드 필드를 주소 필드라고도 부른다.

## 연산 코드
* 연산 코드가 담기는 영역을 연산 코드 필드라고 한다.
* 연산 코드의 종류와 생김새는, CPU마다 다르다.
* 연산 코드는 종류가 많지만 기본적으로 크게 네 가지로 나눌 수 있다.

1. 데이터 전송
- MOVE : 데이터를 옮겨라
- STORE : 메모리를 저장하라
- LOAD : 메모리에서 CPU로 데이터를 가져와라
- PUSH : 스택에 데이터를 저장하라
- POP : 스택의 최상단 데이터를 가져와라

2. 산술/논리 연산
- ADD / SUBTRACT / MULTIPLY / DIVIDE : 덧셈 / 뺄셈 / 곱셉 / 나눗셈
- AND / OR / NOT : AND / OR / NOT 연산을 수행하라
- COMPARE : 두 개의 숫자 또는 TURE / FALSE 값을 비교하라

3. 제어 흐름 변경 : 
- JUMP : 특정 주소로 실행 순서를 옮겨라
- HALF : 프로그램의 실행을 멈춰라
- CALL : 되돌아올 주소를 저장한 채 특정 주소로 실행 순서를 옮겨라
- RETURN : CALL을 호출할 때 저장했던 주소로 돌아가라

4. 입출력 제어
- READ : 특정 입출력 장치로부터 데이터를 읽어라
- WRITE : 특정 입출력 장치로 데이터를 써라
- START IO : 입출력 장치를 시작하라

## 주소 지정 방식
* 오퍼랜드 필드에 데이터가 저장될 위치를 명시할 때, 연산에 사용할 데이터의 위치를 찾는 방법을 의미한다.
* 연산 코드에 사용할 데이터가 저장된 위치, 즉 연산의 대상이 되는 데이터가 저장된 위치를 **유효 주소**라고 한다.

### 주소 지정 방식의 종류
- 즉시 주소 지정 방식 : 연산에 사용할 데이터를 직접 명시
- 직접 주소 지정 방식 : 유효 주소(메모리 주소)를 명시
- 간접 주소 지정 방식 : 유효 주소의 주소를 명시
- 레지스터 주소 지정 방식 : 레지스터에 데이터를 저장한 후 레지스터 이름을 오퍼랜드에 명시
- 레지스터 간접 주소 지정 방식 : 유효 주소(메모리 주소)를 저장한 레지스터 이름을 오퍼랜드에 명시
